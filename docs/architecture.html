terraform-aws-cloudfront-cdn Architecture

HCL / Terraform AWS Infrastructure / IaC

Executive Summary

This module deploys a production-ready Amazon CloudFront CDN distribution with multi-origin support (S3 and ALB), Origin Access Control for secure S3 access, custom cache policies with Brotli/Gzip compression, security response headers (HSTS, X-Frame-Options, XSS Protection), CloudFront Functions and Lambda@Edge for edge compute, WAF integration, geo restrictions, custom error responses for SPA routing, Route 53 DNS alias records, and S3 access logging. It enables organizations to deliver content globally with low latency, strong security headers, and edge-level programmability.

Data Flow

  1. User's browser resolves the custom domain via Route 53, which returns the CloudFront distribution domain as an alias.
  2. The request hits the nearest CloudFront edge location (POP).
  3. AWS WAF (if configured) inspects the request for DDoS, bot, and injection attacks.
  4. Geo restriction rules check the user's country against the whitelist/blacklist.
  5. CloudFront Function executes on viewer-request for URL rewrites, redirects, or header manipulation.
  6. The cache policy checks if a cached response exists (based on TTL, headers, cookies, query strings).
  7. On cache miss, CloudFront forwards to the origin: S3 via OAC (SigV4 signed) or ALB via HTTPS (TLS 1.2).
  8. Lambda@Edge functions (if configured) can transform origin requests/responses.
  9. The response headers policy injects security headers (HSTS, X-Content-Type-Options, X-Frame-Options, X-XSS-Protection) and CORS headers.
  10. For 403/404 errors, custom error responses serve the SPA index.html with a 200 status code.
  11. The response is cached at the edge and returned to the user with Brotli or Gzip compression.
  12. Access logs are written to S3 for analysis.

Security Controls

  • Origin Access Control (OAC) — SigV4-signed requests replace legacy OAI, ensuring only CloudFront can access the S3 origin.
  • TLS 1.2 Enforcement — Minimum protocol version TLSv1.2_2021 for viewer connections; TLS 1.2 for ALB origins.
  • HTTPS Redirect — viewer_protocol_policy = "redirect-to-https" forces all traffic to HTTPS.
  • HSTS Headers — Configurable Strict-Transport-Security with preload and includeSubdomains.
  • X-Frame-Options — Default DENY prevents clickjacking attacks.
  • X-Content-Type-Options — nosniff header prevents MIME type sniffing.
  • XSS Protection — X-XSS-Protection header with mode=block.
  • WAF Integration — AWS WAF Web ACL for DDoS mitigation, bot management, and IP reputation.
  • Geo Restriction — Whitelist or blacklist countries for content access.
  • CORS Policy — Configurable allowed origins, methods, and headers.
  • IPv6 Enabled — Dual-stack support for modern network compatibility.

Production Readiness Checklist

  • Configure custom domain names with ACM certificate (acm_certificate_arn)
  • Set up Route 53 alias records (route53_zone_id)
  • Enable WAF Web ACL (waf_web_acl_id) for DDoS and bot protection
  • Configure response headers policy with security headers (HSTS, X-Frame-Options)
  • Enable access logging (enable_logging = true) for audit trails
  • Set appropriate cache TTLs for your content type (static vs. dynamic)
  • Configure custom error responses for SPA routing (403/404 to index.html)
  • Use PriceClass_100 or PriceClass_200 to control costs by region
  • Configure geo restrictions if content is region-specific
  • Use OAC (not OAI) for S3 origin access -- module uses OAC by default
  • Enable Brotli and Gzip compression (enabled by default in cache policy)
  • Test custom error responses and CloudFront Functions before production

Deployment

Quick Start

module "cloudfront_cdn" {
  source = "github.com/kogunlowo123/terraform-aws-cloudfront-cdn"

  project_name = "my-website"
  environment  = "prod"

  s3_origin_bucket = {
    bucket_regional_domain_name = aws_s3_bucket.website.bucket_regional_domain_name
    bucket_id                   = aws_s3_bucket.website.id
  }

  domain_names        = ["cdn.example.com"]
  acm_certificate_arn = aws_acm_certificate.cdn.arn
  route53_zone_id     = aws_route53_zone.main.zone_id
}
